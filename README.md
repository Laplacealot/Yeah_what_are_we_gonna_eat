# **Yeah_what_are_we_gonna_eat**
# 大模型Agent外卖推荐系统 | 是啊吃什么

## 项目介绍

本项目为个人开发，核心目的是解决“外卖吃什么”的选择困难问题，通过大模型Agent分析个人外卖订单偏好和附近商家信息，生成个性化外卖店铺推荐，帮助快速做出点餐决策。

项目文件说明

项目包含以下核心文件，各文件功能明确，需按指定顺序运行：

- image_splitter_order.py：长截图分割工具，用于分割近期外卖订单长截图（orderlist.jpg），生成可解析的短图

- image_splitter.py：长截图分割工具，用于分割附近商家信息长截图（shop.jpg），生成可解析的短图

- main.py：订单信息解析脚本，运行后生成orders.json文件，记录用户近期外卖订单详细内容

- shop_parser.py：商家信息解析脚本，运行后生成nearby_shops.json文件，记录附近商家详细信息

- user_profile.py：用户画像分析脚本，基于orders.json文件，分析用户的外卖偏好（如口味、价格、品类等）

- recommender.py：外卖推荐脚本，结合用户画像和附近商家信息，推荐3个合适的外卖店铺，并说明推荐理由

## 使用前提

在运行项目前，需完成以下准备工作：

1. 将项目所有代码文件保存至同一文件夹（以下简称“项目根目录”）。

2. 在项目根目录下，添加两个截图文件：
        
  - orderlist.jpg：近期外卖订单的截图（可为长截图）

  - shop.jpg：附近商家信息的截图（可为长截图）
    
  - 项目调用大模型接口，**需要在项目根目录手动新建 .env 文件**，存放大模型 API 密钥与相关配置。

3. 确保已安装Python 3.8及以上版本，以及项目所需的相关依赖包（可通过pip freeze > requirements.txt生成依赖清单，再执行pip install -r requirements.txt安装）。

## 使用步骤

请按照以下顺序运行脚本，否则会导致文件缺失、解析失败等问题：

1. 运行image_splitter_order.py：分割orderlist.jpg长截图，生成可解析的短图，用于后续订单信息提取。

2. 运行image_splitter.py：分割shop.jpg长截图，生成可解析的短图，用于后续商家信息提取。

3. 运行main.py：解析分割后的订单截图，生成orders.json文件，该文件将记录用户近期订单的菜品、价格、店铺等详细信息。

4. 运行shop_parser.py：解析分割后的商家截图，生成nearby_shops.json文件，该文件将记录附近商家的名称、品类、评分、价格等详细信息。

5. 运行user_profile.py：读取orders.json文件，分析用户的外卖偏好（如常点品类、偏好价格区间、高频点餐时段等）。

6. 运行recommender.py：结合user_profile.py分析出的用户画像，以及nearby_shops.json中的商家信息，推荐3个最符合用户偏好的外卖店铺，并详细说明每个店铺的推荐理由。

## 注意事项

- 截图要求：orderlist.jpg和shop.jpg需清晰可见，确保文字、店铺信息、订单详情无遮挡，否则会影响解析效果。

- 文件位置：所有脚本文件、截图文件需放在同一文件夹，不可随意更改文件名称，否则会导致脚本运行失败。

- 运行顺序：必须先运行两个截图分割脚本，再依次运行解析、画像、推荐脚本，不可颠倒顺序。

- 依赖包：若运行过程中出现“模块缺失”错误，需先安装对应的依赖包（可根据错误提示安装，或提前生成并安装requirements.txt中的依赖）。
